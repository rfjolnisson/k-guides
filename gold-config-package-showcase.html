Understanding Package Types

Kaptio supports multiple package types to model your tour products. The key distinctions are: how tours combine, whether tours can sell independently, and whether flights are included.

The Core Definitions

Principle Package
A standalone tour (single or multi-destination) that can be sold independently
Examples: Japan Discovery, South American Discovery
Combo Package
Two or more Principle Tours combined into one product
Example: South Korea tour + Japan tour = Combo
Linked Package
One Principle Tour + one Sub Tour (both in same package)
Example: Egypt (Principle) + Jordan (Sub) = Linked
Extension Tour
Tour Service that CANNOT be booked separately (must add to base tour)
Example: Alexandria Extension (only bookable with Egypt)

Your Package Type Decision Matrix

Package Type Definition Real Example
Principle (Land-Only) Standalone tour, flights not included Japan Discovery, South American Discovery
Principle (Air+Land) Standalone tour, flights included Japan Discovery with Flights
Combo (Land-Only) 2+ Principle Tours combined, no flights South Korea + Japan
Combo (Air+Land) 2+ Principle Tours combined, with flights South Korea + Japan with Flights
Linked Package 1 Principle Tour + 1 Sub Tour Egypt (Principle) + Jordan (Sub)
Extension Tour Tour Service that cannot book separately Alexandria Extension (Egypt only)

Note: Principle Tours can be single or multi-destination. The defining factor is whether it's a standalone tour or a combination of tours.

‚ú® New Feature: Bed Type Defaulting Across Segments

When Service Level Mode = "Package Levels," room type selection cascades across BOTH tour segments automatically.

How It Will Work:
1. Customer selects "Twin Room" on South Korea segment
2. System automatically applies "Twin Room" to Japan segment
3. Selection also applies to Pre-Stay Seoul and Post-Stay Osaka
4. Bridge Hotel (Tokyo) also gets Twin assignment
üö® Important: Sold-Out Behavior

If Twin rooms are sold out on any component (Korea hotels, Japan hotels, bridge hotel, pre-stay, or post-stay), the entire Twin selection is blocked across the whole tour.

No auto-fallback to Single. Customer must explicitly choose Double/Single or join the waitlist. This prevents surprise price increases‚Äînobody wants to book as Twin roommates then discover one hotel changed them to Single rooms.

üìÖ Feature Status: This bed type defaulting capability for combo packages is being implemented as part of the Package Levels feature. Configure Service Level Mode = "Package Levels" to enable cross-segment room type cascading.

Package Type 7

Extension Tours

The Extension Model

Extension Tours are different from Linked Packages. An Extension Tour is a Tour Service that cannot be booked separately‚Äîit must be added to a specific base tour.

Linked Package (Jordan)

‚Ä¢ Jordan is a Sub Tour in the Linked package
‚Ä¢ Can be booked with Egypt or standalone (separate package exists)
‚Ä¢ Customer chooses at booking time
‚úì Jordan CAN sell separately

Extension Tour (Alexandria)

‚Ä¢ Alexandria is an Extension Tour Service
‚Ä¢ CANNOT be booked separately
‚Ä¢ Must be added to Egypt base tour
‚úó Alexandria CANNOT sell alone

Real-World Example: Alexandria Extension

Your Egypt tour includes Cairo, Luxor, and Aswan (9 days). Some customers want to add Alexandria (2 days). But Alexandria doesn't make sense as a standalone product‚Äîit's too short and requires Egypt as context.

How It Appears in Booking:

Base Tour: Highlights of Egypt (9 days, $4,000)
Available Extension: Add Alexandria (+2 days, +$800)
Alexandria only appears as an option when booking Egypt‚Äîit's not searchable as a standalone tour.
Configuration:

Extension Tours are configured as Tour Services with a flag marking them as "extension-only." They appear in the booking wizard as add-ons to their base tour but don't show up in package search.

Extension vs Linked vs Combo: Quick Reference

Combo Package: 2+ Principle Tours combined (e.g., Korea + Japan). Both tours can sell separately.
Linked Package: 1 Principle Tour + 1 Sub Tour (e.g., Egypt + Jordan). Jordan is a Sub Tour in the package.
Extension Tour: Tour Service that CANNOT book separately (e.g., Alexandria). Must be added to base tour.

üîë Understanding Inventory Drivers

Per Pax (Per Person Allotment)

Tour Inventory (Per Pax) means each passenger consumes 1 unit of capacity. If you have 20 capacity and book 12 passengers, you have 8 units remaining‚Äîregardless of room configuration.

Example: 12 passengers book (6 singles, 3 couples in doubles)
Per Pax inventory: 12 units consumed (one per person)
Room inventory: 9 rooms consumed (6 singles + 3 doubles)
Per Pax tracks people count. Room inventory tracks physical rooms. Both work together.
Room Allotment

Room Configuration uses Room-based inventory. A Twin/Double room consumes 1 room unit, regardless of how many people occupy it.

üí° Advanced: Room Type-Specific Inventory

You can control inventory separately for each room type. Useful when you have limited singles.

Example:
‚Ä¢ Total capacity: 20 passengers (Per Pax)
‚Ä¢ Room inventory: 5 singles, 15 doubles/twins
Result: Singles sell out after 5 bookings. Doubles still available for remaining 15 pax capacity.

Release Tiers: Minimize Unused Inventory Risk

You've contracted 20 rooms, but what if you only sell 12? You're paying for 8 empty rooms. Release tiers let you progressively release unused capacity back to the supplier.

Example: 20-room block, December 15 departure
60 days out (Oct 16): Release 5 rooms if unsold ‚Üí 15 remain
30 days out (Nov 15): Release 5 more ‚Üí 10 remain
14 days out (Dec 1): Release 5 more ‚Üí 5 remain
7 days out (Dec 8): Release final 5 ‚Üí 0 contracted
You only pay for rooms you couldn't sell AND didn't release in time. Smart contract management.

Common Questions

Should I create separate packages for Land-Only and Air+Land?

Yes. Create two package records.

Why: Some customers use frequent flyer points and want land-only. Others want all-inclusive pricing. Most operators see 60% book air+land, 40% book land-only. Give them both options.

Can I change minimum capacity after departures exist?

Yes, but do it carefully.

Changing minimum from 10 to 12 means departures with 11 bookings will lose their "Guaranteed" status. Make changes before the season starts, not mid-season.

How do I handle single supplements?

Configure pricing for each room type.

Twin/Double Room

For 2 people (total price)
= $8,787 per person

$10,160 AUD
For 1 person (package + supplement)

How it works: Twin/Double price covers 2 people combined. Single price = full package cost for solo traveler + supplement for solo room occupancy.

What if inventory is less than my maximum capacity?

System uses the lower value.

If you set maximum capacity = 20 but only have 15 rooms available, departure shows "15 spaces left." Inventory trumps capacity setting.

Can I offer the same tour style in different regions?

Yes. Create separate packages, reuse components where possible.

"Discovery Tour" format in Japan, Egypt, Peru‚Äîsame tour style, different destinations. Create 3 packages, but payment schedules, cancellation policies, and some component types can be shared across all three.
